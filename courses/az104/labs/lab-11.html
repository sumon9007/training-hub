Lab 11 🟡 Intermediate ⏱ 45 min

Implement Monitoring

Configure Azure Monitor, Log Analytics, and Application Insights

Module 11 — Monitoring

Lab Objectives

  • ✓Create a Log Analytics workspace and connect resources
  • ✓Configure diagnostic settings to stream logs and metrics
  • ✓Create Azure Monitor alert rules with action groups
  • ✓Enable Application Insights on a web app and review telemetry

Prerequisites

  • •Azure subscription with Monitoring Contributor role
  • •A running VM and a web app in resource group az104-rg11

Lab Steps

Architecture Diagram
1

Create a Log Analytics workspace

Portal

Search for Log Analytics workspaces → + Create. Set Resource group az104-rg11, Name az104-law1, Region East US. Click Review + create → Create.

2

Configure VM diagnostic settings

Portal

Open your VM → Monitoring → Diagnostic settings. Click + Add diagnostic setting. Select all metric categories and send to az104-law1.

3

Enable VM Insights

Portal

Open az104-vm1 → Monitoring → Insights → Enable. Select workspace az104-law1 and click Enable. The Dependency agent and Log Analytics agent will be installed automatically.

4

Create an alert rule

Portal

Search for Monitor → Alerts → + Create → Alert rule. Set Scope to az104-vm1. Under Condition, select signal Percentage CPU, operator Greater than, threshold 80, aggregation granularity 5 minutes. Create an Action group for email notification. Name the rule High-CPU-Alert and click Review + create → Create.

5

Enable Application Insights

Portal

Open your web app → Monitoring → Application Insights → Turn on Application Insights. Link to a new App Insights resource. Enable and save.

6

Run a Log Analytics query

Portal

Open az104-law1 → Logs. Run a KQL query to view VM performance:

// VM CPU over last hour
Perf
| where TimeGenerated > ago(1h)
| where ObjectName == 'Processor'
| where CounterName == '% Processor Time'
| summarize avg(CounterValue) by Computer, bin(TimeGenerated, 5m)
| render timechart

Validation Checklist

  • ✓Log Analytics workspace shows ingested data (Heartbeat table) from the VM.
  • ✓Alert rule High-CPU-Alert appears in Azure Monitor → Alerts → Alert rules.
  • ✓Application Insights live metrics and dependency map load for the web app.
  • ✓KQL query returns results for Processor performance.

Cleanup Resources

Run after the lab to avoid ongoing charges.

az monitor metrics alert delete --name 'High-CPU-Alert' --resource-group az104-rg11
az group delete --name az104-rg11 --yes --no-wait
