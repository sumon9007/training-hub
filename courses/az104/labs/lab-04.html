Lab 04 🟡 Intermediate ⏱ 40 min

Implement Virtual Networking

Create VNets, subnets, NSGs, and configure Azure DNS settings

Module 4 — Configure and Manage Virtual Networks

Lab Objectives

  • ✓Create a virtual network with multiple subnets
  • ✓Create and associate a Network Security Group
  • ✓Configure NSG inbound and outbound security rules
  • ✓Configure Azure private DNS and link it to a VNet

Prerequisites

  • •Azure subscription with Contributor role
  • •Resource group az104-rg4 in East US

Lab Steps

Architecture Diagram
1

Create a virtual network

Portal

Search for Virtual networks → + Create. Set Resource group az104-rg4, Name az104-vnet1, Region East US. Under IP Addresses, set address space to 10.40.0.0/16. Click Review + create → Create.

2

Add subnets

Portal

Open az104-vnet1 → Subnets → + Subnet. Add subnet0 with address range 10.40.0.0/24. Click Save. Add another subnet subnet1 with 10.40.1.0/24. Click Save.

3

Create an NSG and rules

Portal

Search for Network security groups → + Create. Set Resource group az104-rg4, Name az104-nsg1, Region East US. After creation, open az104-nsg1 → Inbound security rules → + Add. Set Destination port 3389, Protocol TCP, Action Allow, Priority 100, Name AllowRDP. Click Add.

4

Associate the NSG to subnet0

Portal

Open az104-nsg1 → Subnets → + Associate. Select Virtual network az104-vnet1 and Subnet subnet0. Click OK.

5

Create an Azure DNS private zone

Portal

Search for Private DNS zones → + Create. Set Resource group az104-rg4, Name private.contoso.com. After creation, click + Virtual network link → + Add. Enter link name az104-vnet1-link, select VNet az104-vnet1, enable Auto registration. Click OK.

6

Verify the configuration

Portal

Open az104-vnet1 in the portal. Confirm subnets, check Connected devices, and open DNS servers to see the private zone link.

Validation Checklist

  • ✓VNet az104-vnet1 has two subnets (10.40.0.0/24 and 10.40.1.0/24).
  • ✓NSG az104-nsg1 is associated with subnet0.
  • ✓Private DNS zone private.contoso.com is linked to az104-vnet1 with auto-registration.

Cleanup Resources

Run after the lab to avoid ongoing charges.

az group delete --name az104-rg4 --yes --no-wait
