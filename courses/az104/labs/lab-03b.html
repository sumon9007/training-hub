Lab 03b 🟡 Intermediate ⏱ 30 min

Resources via ARM Templates

Deploy and manage Azure resources using ARM JSON templates

Module 3 — Manage Azure Resources

Lab Objectives

  • ✓Export an ARM template from an existing resource
  • ✓Modify template parameters for a new deployment
  • ✓Deploy resources from an ARM template via portal and CLI
  • ✓Understand template schema structure and linked templates

Prerequisites

  • •Azure subscription with Contributor role
  • •Resource group az104-rg1 (or create it during the lab)

Lab Steps

Architecture Diagram
1

Create a resource to export

Portal

Create a simple storage account named az104armtest[unique] in az104-rg1 via the portal.

2

Export the ARM template

Portal

Open the storage account, click Export template in the sidebar. Review the JSON structure — note parameters, variables, and resources sections. Click Download.

3

Modify the template

Portal

Open the downloaded template.json in a text editor. In the parameters section, update the storageAccountName default value to a unique name such as az104arm2[5-digit-suffix]. Save the file.

4

Deploy via Custom Template

Portal

In the portal, search for Deploy a custom template. Click Build your own template in the editor, then Load file and select your modified template.json. Click Save, fill in az104-rg1 as the resource group and review parameters. Click Review + create → Create.

5

Monitor the deployment

Portal

In az104-rg1, click Deployments. Select arm-lab-deployment to see the deployment timeline, operations, and outputs.

6

Use a Quickstart template

Portal

Navigate to Deploy a custom template in the portal. Click Load a GitHub quickstart template, search for 101-storage-account-create, and deploy it to az104-rg1.

Validation Checklist

  • ✓Two storage accounts exist in az104-rg1.
  • ✓Deployment arm-lab-deployment shows Succeeded in the Deployments blade.

Cleanup Resources

Run after the lab to avoid ongoing charges.

az group delete --name az104-rg1 --yes --no-wait
