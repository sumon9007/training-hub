Lab 09c 🔴 Advanced ⏱ 40 min

Azure Kubernetes Service

Create an AKS cluster, deploy applications, and scale workloads

Module 9 — PaaS Compute Options

Lab Objectives

  • ✓Create an AKS cluster with Azure CNI networking
  • ✓Deploy an application using kubectl and a YAML manifest
  • ✓Expose the application with a Kubernetes LoadBalancer service
  • ✓Scale the deployment and configure Horizontal Pod Autoscaler

Prerequisites

  • •Azure subscription with Contributor role
  • •Azure CLI with aks-preview extension
  • •kubectl installed or use Cloud Shell (it's pre-installed)

Lab Steps

Architecture Diagram
1

Create an AKS cluster

Portal

Search for Kubernetes services → + Create → Create a Kubernetes cluster. Set Resource group az104-rg9c, cluster name az104-aks1, Region East US, Node size Standard_B2s, Node count 2, Network plugin Azure CNI. Enable Managed identity. Click Review + create → Create (takes 4–6 min).

2

Get cluster credentials

Portal

Open az104-aks1 → Overview. Click Connect to see the credentials command. In Cloud Shell, run the displayed az aks get-credentials command, then run kubectl get nodes to verify all nodes show Ready.

3

Deploy an application

Portal

In Cloud Shell, run: kubectl create deployment az104-nginx --image=nginx:latest --replicas=2 followed by kubectl get pods -o wide to see pods scheduled across nodes.

4

Expose via LoadBalancer service

Portal

In Cloud Shell, run: kubectl expose deployment az104-nginx --type=LoadBalancer --port=80 then kubectl get svc az104-nginx --watch to wait for an external IP (2–3 min). Open the IP in a browser to verify nginx responds.

5

Scale the deployment

Portal

In az104-aks1 → Workloads, find the az104-nginx deployment. In Cloud Shell run: kubectl scale deployment az104-nginx --replicas=4 and kubectl autoscale deployment az104-nginx --cpu-percent=50 --min=2 --max=10 to configure HPA.

6

Monitor with Azure Monitor for containers

Portal

In the portal, open az104-aks1 → Insights. Explore the Cluster, Nodes, and Pods views. Review container CPU and memory metrics.

Validation Checklist

  • ✓All cluster nodes show Ready status in kubectl get nodes.
  • ✓nginx deployment has 4 running pods.
  • ✓LoadBalancer service has an external IP and returns nginx page.
  • ✓HPA is configured and monitoring the deployment.

Cleanup Resources

Run after the lab to avoid ongoing charges.

kubectl delete deployment az104-nginx
kubectl delete svc az104-nginx
az aks delete --name az104-aks1 --resource-group az104-rg9c --yes --no-wait
