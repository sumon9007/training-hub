Lab 09b 🟡 Intermediate ⏱ 20 min

Azure Container Instances

Deploy containerized workloads using Azure Container Instances

Module 9 — PaaS Compute Options

Lab Objectives

  • ✓Deploy a container from a public Docker image using ACI
  • ✓Configure environment variables and resource limits
  • ✓Inspect container logs and resource utilization
  • ✓Deploy a multi-container group using a YAML definition

Prerequisites

  • •Azure subscription with Contributor role
  • •Resource group az104-rg9b

Lab Steps

Architecture Diagram
1

Deploy a container instance

Portal

Search for Container Instances → + Create. Set Resource group az104-rg9b, container name az104-aci1, image source Other registry, Image nginx:latest, OS Linux. Under Networking, set DNS label (e.g. az104aci[random]), open port 80. Click Review + create → Create.

2

Get the public IP and test

Portal

Open az104-aci1 → Overview. Note the IP address and FQDN in the essentials panel. Open the FQDN in a browser — you should see the nginx welcome page.

3

View container logs

Portal

In az104-aci1, click Containers in the sidebar. Select the container name, then click the Logs tab to view stdout/stderr output.

4

Deploy a multi-container group

Portal

Search for Container Instances → + Create. On the Advanced tab, select YAML as the deployment option. Paste or upload a container group YAML that defines a web container (nginx:alpine, port 80) and a sidecar container (busybox). Review and create.

5

Monitor with Azure Monitor

Portal

Open az104-aci1 in the portal. Click Monitoring → Metrics to view CPU and memory utilization charts.

Validation Checklist

  • ✓ACI container returns nginx welcome page on port 80.
  • ✓Container logs show HTTP request entries after the curl test.
  • ✓Multi-container group shows both web and sidecar containers running.

Cleanup Resources

Run after the lab to avoid ongoing charges.

az container delete --name az104-aci1 --resource-group az104-rg9b --yes
az container delete --name az104-aci-group --resource-group az104-rg9b --yes
